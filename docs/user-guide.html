1. Sebastian 소개

1.1 Sebastian이란?

Sebastian은 3개 게임(MIR4, NC, LY Table)의 현지화 테이블 관리 도구를 하나로 통합한 프로그램입니다.

✅ 주요 장점
  • 하나의 프로그램으로 3개 게임 관리
  • 직관적인 UI: 탭 방식으로 게임 선택
  • 실시간 진행 상황: 작업 진행률 및 현재 파일명 표시
  • 레거시 기능 100% 보장: 기존 스크립트와 동일한 결과
  • 빠른 성능: 병렬 처리로 속도 향상 (NC/GL)

1.2 지원 게임 및 기능

게임 기능 설명
M4/GL
(MIR4)		 DIALOGUE 병합
STRING 병합		 3개 파일 → 1개 통합
8개 파일 → 1개 통합
NC/GL
(NC)		 8개 언어 병합 언어별 8개 파일 → 1개 통합
날짜/마일스톤 입력
LY/GL
(LY Table)		 Merge, Split
Batches, Diff
Status Check		 7개 언어 ⇄ 1개 통합
배치 병합, 버전 비교
상태 검증

2. 설치 및 실행

2.1 설치

1 Sebastian.exe 다운로드
  • GitHub Releases에서 최신 버전 다운로드
  • URL: https://github.com/JaekyungCho2140/sebastianmk2/releases
  • 파일 크기: 약 89MB
2 원하는 폴더에 복사
  • 별도 설치 불필요 (단일 실행 파일)
  • 예: C:\Tools\Sebastian\Sebastian.exe
3 실행
  • Sebastian.exe 더블 클릭
  • Windows Defender 경고 시: "추가 정보" → "실행" 클릭
📷 스크린샷 위치: Sebastian.exe 실행 화면 (메인 창)

2.2 메인 화면 구성

📷 스크린샷 위치: 메인 창 전체 (탭 3개 표시)

화면 구성:

  • 상단: 메뉴바 (파일, 도움말)
  • 탭바: M4/GL, NC/GL, LY/GL 선택
  • 메인 영역: 각 탭의 기능 버튼 및 입력 필드
  • 하단: 로그 뷰어 (펼치기/접기 가능)
  • 상태바: 현재 작업 상태 표시

3. M4/GL 사용법 (MIR4)

3.1 DIALOGUE 병합

📝 목적: 3개 대화 파일을 1개로 통합
📂 입력 파일: CINEMATIC_DIALOGUE.xlsm, SMALLTALK_DIALOGUE.xlsm, NPC.xlsm
📄 출력 파일: MERGED_DIALOGUE.xlsx

단계별 사용법

1 M4/GL 탭 선택
  • 메인 창 상단의 "M4/GL" 탭 클릭
📷 스크린샷 위치: M4/GL 탭 화면 (DIALOGUE/STRING 카드 버튼)
2 DIALOGUE 카드 클릭
  • 왼쪽 카드 "DIALOGUE 병합" 클릭
  • 카드가 보라색으로 강조됨 (선택 상태)
3 폴더 선택
  • "📁 폴더 선택" 버튼 클릭
  • 3개 파일이 있는 폴더 선택
  • 경로가 입력 필드에 표시됨
4 실행
  • "실행 (Enter)" 버튼 클릭 (또는 Enter 키)
  • 진행 상황 Dialog 표시
📷 스크린샷 위치: 진행 상황 Dialog (진행바, 파일명 표시)
5 완료 확인
  • "완료" 메시지 표시
  • 출력 파일: 선택한 폴더에 MERGED_DIALOGUE.xlsx 생성
✅ 성공 시: "M4/GL DIALOGUE 병합 완료: [폴더 경로]" 메시지 표시

3.2 STRING 병합

📝 목적: 8개 문자열 파일을 1개로 통합
📂 입력 파일: SEQUENCE_DIALOGUE.xlsm, STRING_BUILTIN.xlsm, ...(8개)
📄 출력 파일: MERGED_STRING.xlsx

사용법: DIALOGUE와 동일하지만 "STRING" 카드 선택

⚠️ 주의사항:
  • 8개 파일이 모두 폴더에 있어야 합니다
  • 파일명은 정확히 일치해야 합니다 (대소문자 구분)

4. NC/GL 사용법 (NC)

4.1 8개 언어 병합

📝 목적: 8개 언어 파일을 1개로 통합
📂 입력 파일: StringEnglish.xlsx, StringTraditionalChinese.xlsx, ...(8개)
📄 출력 파일: {날짜}_M{마일스톤}_StringALL.xlsx (예: 250512_M15_StringALL.xlsx)

단계별 사용법

1 NC/GL 탭 선택
📷 스크린샷 위치: NC/GL 탭 화면 (날짜/마일스톤 입력 필드)
2 날짜 입력
  • YYMMDD 형식 (6자리 숫자)
  • 예: 250512 (2025년 5월 12일)
  • 유효하면 초록색 체크(✓) 표시
3 마일스톤 입력
  • 1-3자리 숫자
  • 예: 15 → M15로 저장
  • 유효하면 초록색 체크(✓) 표시
4 폴더 선택
  • "폴더 선택" 버튼 클릭
  • 8개 파일이 있는 폴더 선택
5 실행
  • "실행 (Enter)" 버튼 클릭
  • 진행 상황 Dialog 표시 (8개 파일 병렬 처리)
✅ 성공 시: "NC/GL 병합 완료: [폴더 경로] (날짜: 250512, 마일스톤: M15)" 메시지
⚠️ 주의사항:
  • 날짜는 정확히 6자리 숫자여야 합니다
  • 마일스톤은 1-3자리 숫자만 입력 가능 (M은 자동 추가)

5. LY/GL 사용법 (LY Table)

LY/GL은 5개 기능을 제공합니다. 각 기능은 수직 리스트 버튼으로 표시됩니다.

📷 스크린샷 위치: LY/GL 탭 화면 (5개 버튼 리스트)

5.1 Merge (병합)

📝 목적: 7개 언어 파일 → 1개 통합 파일
📂 입력 파일: EN.xlsx, CT.xlsx, CS.xlsx, JA.xlsx, TH.xlsx, PT-BR.xlsx, RU.xlsx
📄 출력 파일: 사용자 지정 경로

단계별 사용법

1 LY/GL 탭 → "Merge" 버튼 클릭
📷 스크린샷 위치: Merge Wizard Dialog
2 7개 파일 선택
  • "파일 추가" 버튼으로 7개 파일 선택
  • 또는 폴더에서 자동 스캔
3 출력 경로 선택
  • "출력 파일" 버튼 클릭
  • 저장할 파일명 및 경로 지정
4 "시작" 버튼 클릭
  • 진행 상황 Dialog 표시
  • 완료 시 출력 파일 생성

5.2 Split (분리)

📝 목적: 1개 통합 파일 → 7개 언어 파일
📂 입력 파일: 통합 파일 (Merge 형식)
📄 출력 파일: EN.xlsx, CT.xlsx, ...(7개)

단계별 사용법

1 LY/GL 탭 → "Split" 버튼 클릭
2 통합 파일 선택
  • Merge로 생성한 파일 선택
3 출력 폴더 선택
  • 7개 파일을 저장할 폴더 선택
4 "시작" 버튼 클릭
✅ Round-trip 보장: Merge → Split → Merge 시 원본과 100% 일치

5.3 Batches (배치 병합)

📝 목적: 여러 배치 병합 + Status 자동 완료 처리
📂 입력: Root Folder (배치 폴더들)
📄 출력: merged_batches.xlsx

단계별 사용법

1 LY/GL 탭 → "Batches" 버튼 클릭
📷 스크린샷 위치: Batch Wizard Dialog (배치 리스트)
2 Root Folder 선택
  • 배치 폴더들이 있는 상위 폴더 선택
  • 자동으로 배치 스캔 및 표시
3 배치 선택
  • 병합할 배치들 체크박스 선택
  • Base Batch 선택 (기준 배치)
4 Auto Complete 옵션
  • 체크 시: 모든 Target 값이 채워진 KEY → Status = "완료"
5 출력 경로 선택 및 시작

5.4 Diff (버전 비교)

📝 목적: 두 버전 비교 → 변경 사항 추적
📂 입력: 폴더 1, 폴더 2 (각 7개 언어 파일)
📄 출력: diff_report.xlsx (Added/Deleted/Modified 시트)

단계별 사용법

1 LY/GL 탭 → "Diff" 버튼 클릭
2 폴더 1 선택 (이전 버전)
3 폴더 2 선택 (현재 버전)
4 출력 경로 선택 및 시작
✅ 결과:
  • Sheet1: Added - 새로 추가된 KEY (초록색)
  • Sheet2: Deleted - 삭제된 KEY (빨간색)
  • Sheet3: Modified - 변경된 KEY (노란색)
  • Sheet4: Summary - 통계

5.5 Status Check (상태 검증)

📝 목적: 7개 언어 파일의 Status 일치 검증
📂 입력: 7개 언어 파일
📄 출력: status_check_report.xlsx

단계별 사용법

1 LY/GL 탭 → "Status Check" 버튼 클릭
2 7개 파일 선택
3 출력 경로 선택 및 시작
✅ 결과:
  • Sheet1: Mismatches - 불일치 KEY 목록
  • Sheet2: Statistics - 전체/일치/불일치 통계
  • Sheet3: Language Summary - 언어별 Status 분포

6. 문제 해결

6.1 자주 발생하는 오류

❌ "파일을 찾을 수 없습니다"

원인: 필요한 파일이 폴더에 없음

해결:

  • 폴더 경로 확인
  • 파일명 확인 (대소문자 정확히 일치해야 함)
  • 필요한 개수만큼 파일 존재 확인 (M4/GL DIALOGUE: 3개, STRING: 8개, NC/GL: 8개, LY/GL: 7개)
❌ "헤더가 올바르지 않습니다"

원인: Excel 파일 구조가 예상과 다름

해결:

  • 원본 파일 확인 (레거시 스크립트 출력과 비교)
  • 파일 손상 여부 확인
  • 올바른 파일인지 재확인
❌ "KEY가 일치하지 않습니다" (LY/GL)

원인: 7개 언어 파일의 KEY가 다름

해결:

  • EN 파일 기준으로 다른 언어 파일 확인
  • 누락된 KEY 추가 또는 불필요한 KEY 제거

6.2 실행 관련 문제

❌ "Windows Defender 차단"

해결:

  1. "추가 정보" 클릭
  2. "실행" 버튼 클릭
  3. (선택) Windows Defender 제외 목록에 추가
❌ "프로그램이 느림"

원인: 대용량 파일 처리

해결:

  • 진행 상황 Dialog에서 현재 상태 확인
  • NC/GL은 병렬 처리로 속도 향상됨
  • 작업 완료까지 대기 (취소 버튼으로 중단 가능)

7. 자주 묻는 질문 (FAQ)

Q1. 레거시 스크립트와 결과가 다른가요?

A: 아니요. Sebastian은 레거시 로직을 100% 보존하여 출력 파일이 완전히 동일합니다. (M4/GL, NC/GL은 레거시 출력과 비교 검증 완료, LY/GL은 37개 Round-trip 테스트 통과)

Q2. 여러 작업을 동시에 할 수 있나요?

A: 아니요. 한 번에 하나의 작업만 실행 가능합니다. 진행 중인 작업이 완료되면 다음 작업을 시작할 수 있습니다.

Q3. 로그는 어디서 확인하나요?

A: 메인 창 하단의 로그 뷰어에서 확인 가능합니다.
  • "▼ 로그 보기" 클릭하여 펼치기
  • 메뉴: 파일 → 로그 저장으로 텍스트 파일로 저장 가능

Q4. 작업을 취소할 수 있나요?

A: 예. 진행 상황 Dialog의 "취소" 버튼을 클릭하면 작업이 중단됩니다. (주의: 부분적으로 생성된 파일은 삭제되지 않으므로 수동 삭제 필요)

Q5. NC/GL의 날짜 형식은 어떻게 입력하나요?

A: YYMMDD 형식 (6자리 숫자)입니다.
  • 예: 2025년 5월 12일 → 250512
  • 실시간 검증: 유효하면 초록색 체크(✓), 무효하면 빨간색 X(✗) 표시

Q6. LY/GL Round-trip이란 무엇인가요?

A: Merge → Split → Merge 시 원본과 100% 일치하는 것을 보장합니다.
  • 7개 언어 파일 → 통합 병합 → 7개 분리 → 재병합 = 원본 병합 결과
  • 37개 테스트 케이스로 검증 완료

Q7. 배치(Batches)의 "Auto Complete"는 무엇인가요?

A: 모든 Target_{언어} 값이 채워진 KEY의 Status를 자동으로 "완료"로 변경하는 기능입니다.
  • 체크 시: 7개 언어 모두 번역 완료된 KEY → Status = "완료"
  • 미체크 시: Status 값 유지

Q8. 파일 크기가 큰 이유는?

A: Sebastian.exe는 Python 런타임 및 모든 라이브러리를 포함한 단일 실행 파일입니다.
  • 파일 크기: 약 89MB
  • 최적화 전: 2.7GB → 최적화 후: 89MB (96.7% 감소)
  • 별도 설치 불필요 (장점)

Q9. 다른 게임도 추가할 수 있나요?

A: 향후 계획에 포함되어 있습니다. 현재는 M4/GL, NC/GL, LY/GL 3개 게임만 지원합니다.
